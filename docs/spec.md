# Centaurlog - Project Specification

**Status**: Active Development
**Created**: 2026-02-21

## Description

A blog specifically for LLM-written content with clear AI authorship disclosure. Markdown-based workflow where Claude writes, user reviews/guides, then Claude publishes. Primarily technical/research content for personal reference and sharing via URLs.

## Goals

1. **Low-friction publishing** - Start writing content immediately with minimal setup
2. **Clear AI attribution** - Prominent LLM authorship disclosure throughout site
3. **Shareable URLs** - Easy to point others to specific posts (e.g., for FAQs)
4. **Migration-friendly** - Use portable format (markdown) that can move to other platforms later
5. **RSS/Atom support** - Allow feed subscriptions

## Scope

### In Scope
- Markdown-based writing workflow
- GitHub-based hosting and publishing
- Minimal but functional theme/styling
- RSS/Atom feed generation
- LLM authorship attribution on every page
- Custom domain support (centaurlog.ohuiginn.net) - configuration only, DNS setup later
- Version control for all content
- Claude can write and publish directly via git

### Out of Scope (for MVP)
- Comments system
- Analytics/tracking
- Advanced search functionality
- Multi-author support
- Content management UI beyond markdown files
- Social media integration
- Email newsletter integration

## Technical Approach

### Platform
- **GitHub Pages** with Jekyll (GitHub's built-in static site generator)
- Leverage existing `danohu.github.io` repo structure/style as starting point
- Markdown files for all content
- Git-based publishing workflow

### Repository Structure
```
centaurlog/
├── centaurlog.git/     # Bare repository
└── main/               # Main worktree
    ├── _posts/         # Blog posts in markdown
    ├── _config.yml     # Jekyll configuration
    ├── _layouts/       # Page templates
    ├── _includes/      # Partial templates
    ├── assets/         # CSS, images
    ├── feed.xml        # RSS/Atom feed (auto-generated by Jekyll)
    ├── local/          # Per-worktree state (reserved for future use)
    └── README.md
```

### Workflow
1. User requests post topic
2. Claude writes markdown draft
3. User reviews and provides feedback
4. Iterate until satisfied
5. Claude commits and pushes to publish

### Style/Theme
- Copy/adapt style from existing `danohu.github.io` Jekyll blog as baseline
- Ensure clear visual distinction (different color scheme, header, etc.)
- **Attribution**: Subheading below blog title: "all posts written by LLMs under the guidance of [Dan O'Huiginn](https://ohuiginn.net)"
  - Prominent placement on every page
  - Dan's name linked to https://ohuiginn.net

### Domain
- Initial: `danohu.github.io/centaurlog` (GitHub Pages from new repo)
- Target: `centaurlog.ohuiginn.net` (custom domain configured in GitHub Pages settings)

## Success Criteria
- [ ] Can write markdown post and publish via git push
- [ ] Posts visible at public URL
- [ ] RSS/Atom feed working and discoverable
- [ ] Clear LLM attribution on every page
- [ ] Custom domain configured (even if not yet using it)
- [ ] Style adapted from existing blog provides clean, readable layout
- [ ] Can share individual post URLs that render properly

## Implementation Notes

### Jekyll Basics
- Posts in `_posts/` directory with naming: `YYYY-MM-DD-title.md`
- Frontmatter in YAML:
  ```yaml
  ---
  layout: post
  title: "Post Title"
  date: 2026-02-21
  model: "Claude Sonnet 4.5"
  tags: [tag1, tag2]
  ---
  ```
- RSS/Atom feed auto-generated via `jekyll-feed` plugin (included by default)
